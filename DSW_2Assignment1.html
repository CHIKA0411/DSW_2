b. dictionary vs default dictionary

Dictionary (dict):

A standard Python dictionary is a collection that stores unordered key-value pairs. Each key in a dictionary must be unique. When you try to access a key that doesn't exist in a standard dictionary, it raises a KeyError. Example:

In [5]:
my_dict = {'abha': 1, 'bina': 2, 'ck': 3}
print(my_dict['abha'])  

1

Default Dictionary (collections.defaultdict):

defaultdict is a subclass of the built-in dict class in the collections module. It allows you to provide a default value for each key. When you access a key that doesn't exist, it automatically creates the key and assigns it the default value. This avoids KeyError exceptions that you might encounter when using a standard dictionary. Example:

In [6]:
from collections import defaultdict

# Define a defaultdict with default value as 0
my_defaultdict = defaultdict(int)
my_defaultdict['abha'] = 100
print(my_defaultdict['bina'])  

0

c. args and *kwargs

args: The args parameter allows a function to accept any number of positional arguments. When a function is defined with *args, it can accept zero or more positional arguments. Inside the function, args becomes a tuple containing all the positional arguments passed to the function. Example:

In [7]:
def my_function(*args):
    for arg in args:
        print(arg)

**kwargs:

The kwargs parameter allows a function to accept any number of keyword arguments. When a function is defined with kwargs, it can accept zero or more keyword arguments in the form of key=value pairs. Inside the function, kwargs becomes a dictionary containing all the keyword arguments passed to the function. Example:

In [8]:
def my_function(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

d. Data science vs data engineers

Data science and data engineering are two distinct but closely related fields within the broader domain of data analytics and management. Here's a brief comparison:

  1. Data Science:

    • Focus: Data science primarily focuses on deriving insights and knowledge from data through analysis, modeling, and interpretation.
    • Skills: Data scientists typically possess skills in statistics, machine learning, data visualization, and domain expertise.
    • Responsibilities:
      • Cleaning and preprocessing data.
      • Exploratory data analysis (EDA) to understand data characteristics.
      • Building and training predictive models.
      • Extracting actionable insights from data.
    • Tools: Data scientists commonly use programming languages like Python or R, along with libraries such as pandas, scikit-learn, TensorFlow, and PyTorch.
    • Goal: The goal of data science is to extract value from data by uncovering patterns, making predictions, and informing decision-making processes.

  2. Data Engineering:

    • Focus: Data engineering focuses on the design, construction, and maintenance of data pipelines and infrastructure for large-scale data processing and storage.
    • Skills: Data engineers typically possess skills in software engineering, database design, distributed systems, and cloud computing.
    • Responsibilities:
      • Building and maintaining data infrastructure (data warehouses, data lakes, ETL pipelines).
      • Developing scalable data processing systems.
      • Ensuring data quality and integrity.
      • Collaborating with data scientists and analysts to deploy models and enable data-driven decision-making.
    • Tools: Data engineers use a variety of tools and technologies, including SQL databases, NoSQL databases, distributed computing frameworks (e.g., Apache Spark), containerization (e.g., Docker), and cloud services (e.g., AWS, Azure, Google Cloud Platform).
    • Goal: The goal of data engineering is to provide a robust and efficient infrastructure for collecting, storing, and processing data, enabling data scientists and analysts to work effectively with large volumes of data.

In summary, while data scientists focus on extracting insights and building models to understand and predict phenomena, data engineers focus on building and maintaining the infrastructure necessary to support these activities at scale. Both roles are essential in the data-driven decision-making process, often working collaboratively to extract value from data.

4. Count frequencies of various alphabets (Convert upper case into lower case and input given by user), plot the results for this as a bar chart with x-axis being the letter and y-axis as the corresponding frequency.¶

In [21]:
import matplotlib.pyplot as plt
from collections import defaultdict
word=input("Enter a word : ")
def count_alphabet_frequencies(word):
    lower_case=word.lower()
    frequencies = defaultdict(int)
    
     # Count frequencies
    for char in lower_case:
        if char.isalpha():
            frequencies[char] += 1

    return frequencies
alphabet_frequencies = count_alphabet_frequencies(word)

# Extract data for plotting
letters = list(alphabet_frequencies.keys())
frequencies = list(alphabet_frequencies.values())

# Plotting the results

plt.bar(letters, frequencies)
plt.xlabel('Alphabet')
plt.ylabel('Frequency')
plt.title('Frequency of Alphabets')
plt.show()
